# 后台管理系统使用说明

## 访问后台

1. 打开网站，点击页脚的「🔒 管理后台」链接（或直接访问 `/admin/login.html`）
2. 默认密码：`admin123`（建议登录后立即在「网站设置」中修改）

## 工作流程：草稿 → 预览 → 发布

后台的所有修改（作品、图片、个人信息）会先保存为**本地草稿**，
不会立即影响正式网站。确认无误后点击右上角「🚀 发布到网站」，
修改会通过 GitHub API 提交到仓库，约1-2分钟后全网生效。

- **预览草稿**：查看包含未发布修改的网站效果
- **查看网站**：查看当前已发布的正式网站
- 发布按钮带 ● 标记表示有未发布的修改

## 功能说明

### 🎨 作品管理
- **添加新作品**：上传图片、填写中/英/日标题和描述（英日选填，默认同中文）
- **批量上传**：拖拽多张图片，每张自动创建一个作品（以文件名为标题），之后可逐个编辑
- **编辑 / 删除 / 批量删除**：均为草稿操作，发布后生效
- 图片自动压缩（最长边1600px），单张最大10MB，支持 JPG/PNG/GIF/WebP

### 👤 个人信息
- 姓名（中/英/日）、艺术家照片、个人简介、联系邮箱、统计数字

### ⚙️ 网站设置
- **发布设置**：配置 GitHub Token（首次使用必须配置，详见仓库根目录 README）
- **更改管理密码**
- **导出/导入备份**：下载或恢复完整的内容备份（JSON文件）
- **放弃未发布的修改**：把草稿恢复成正式网站当前的内容

## 安全说明

- 登录密码只是防误操作的前台屏障（纯前端校验）
- 真正的发布权限由 GitHub Token 控制；Token 只保存在配置它的那台浏览器中
- 不要在公共电脑上保存 Token

## 故障排除

- **发布失败「Token 无效」**：Token 过期或被撤销，重新生成并在发布设置中更新
- **发布失败「权限不足」**：确认 Token 授予了本仓库 Contents: Read and write 权限
- **发布成功但网站没变化**：GitHub Pages 部署需要1-2分钟，请稍候并强制刷新（Ctrl+Shift+R）
- **登录失败**：确认密码正确；忘记密码可在登录页点「重置为默认密码」
